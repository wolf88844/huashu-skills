# 长文转社交平台内容

将 3000-5000 字长文浓缩成各平台适配的短内容，保持花生风格。

## 支持平台

| 平台 | 字符数 | 风格 | 开头类型 |
|------|--------|------|----------|
| X/Twitter | 200-500 字 | 口语化、观点犀利、短句为主 | 金句/数据/价值主张 |
| 小红书 | 300-800 字 | 有质感、口语但精致、emoji 丰富 | 痛点共鸣/成果冲击/价值直给 |
| 即刻 | 200-500 字 | 更随意更真实、像日记/碎碎念 | 经历切入/观点抛出 |

## 何时使用

- 用户说"转 X"、"转小红书"、"发即刻"、"生成社交媒体内容"
- 文章完成后需要多平台分发
- 用户提供原文让你改写

## 第一步：确认目标平台

```
用户：帮我把这篇文章发小红书
→ 平台：小红书
→ 字符数：300-800 字
→ 风格：口语精致感 + emoji + 标签
```

## 各平台写法

### X/Twitter 写法

**三种切入角度：**

**A. 金句开头（观点类）**
```
AI时代，永远别低估一个人可能拥有的创造力。

半年前，我在完全没用过unity，没做过游戏开发的情况下尝试过用Cursor+Unity开发军事题材的3D射击游戏...
```

**B. 数据/成果冲击（测试/案例类）**
```
Claude Code是最好的自动化写作agent。

这篇3000多字的文章100%是让Claude Code帮我完成的，在小红书数据不错，也引起了很多讨论...

迟点在评论区分享我的写作agent构建方式
```

**C. 价值主张直给（工具/方法类）**
```
可能是迄今为止最好用的Markdown内容至公众号排版器了，突出体现在省事，以及美观。

几个特点：
1、支持直接复制飞书、notion...
2、内置了10多种从科技媒体、纸媒借鉴的样式...
```

**标准结构（200-500字）：**
1. 开头（1-2句，30-50字）— 金句/数据/价值主张
2. 展开（3-5句或列表，100-300字）— 具体细节/数字/个人经历
3. 结尾（1句，20-30字）— 互动引导

### 小红书写法

**三种切入角度：**

**A. 痛点共鸣开头**
```
为什么你总觉得"等准备好了再说"？

我用了6年才明白：不是准备好了才出发，是出发了才准备好...
```

**B. 成果/数据冲击**
```
这篇3000字的文章，我用AI辅助写了100%，小红书发完评论区炸了。

大家最惊讶的不是内容，是没有人发现这是AI写的...
```

**C. 价值主张直给**
```
如果你也总觉得"再等等我就准备好了"，这句话可能会改变你的2025：

不是准备好了才出发，是出发了才准备好。
```

**标准结构（300-800字）：**

```
1. 封面标题（一句话，戳痛点 or 吊胃口）
2. 开头（2-3句，引起共鸣）
3. 展开（3-5段，故事/经历/方法）
4. 金句/核心观点（单独一行，加粗或框起来的感觉）
5. 结尾互动引导（评论区见/有问题留言）
6. 标签（3-8个 #关键词）
```

**风格要点：**
- ✅ 口语但有质感，不是完全随意
- ✅ 适量 emoji（每段 1-2 个，不要太密）
- ✅ 短句为主，读起来轻松
- ✅ 具体数字、时间线、细节
- ❌ 不要太正式、不要官腔
- ❌ 不要 AI 味

**常用 emoji：**
💡 观点抛出时
🔥 强烈共鸣时
🤔 引发思考时
✨ 总结亮点时
📍 强调地点/时间时
🙋 引导互动时

### 即刻写法

**三种切入角度：**

**A. 经历切入（更像日记）**
```
最近有个感受越来越强：

不是准备好了才出发，是出发了才准备好。

事情是这样的...
```

**B. 观点直给（更犀利）**
```
我见过最害人的思维陷阱之一：

"等我准备好了再做"。

它让人心安理得地躺着，同时焦虑地消耗自己...
```

**C. 一句话+展开（碎碎念感）**
```
"再等等"是这个时代最贵的借口。

说说我的经历...
```

**标准结构（200-500字）：**

```
1. 一句话开头（观点 or 感受，1-2句）
2. 展开叙述（经历/故事/思考，3-6句）
3. 一个小结或金句（1-2句）
4. 互动引导（欢迎讨论/评论区见）
```

**风格要点：**
- ✅ 更随意、更真实，像在和朋友聊天
- ✅ 可以有一点碎碎念
- ✅ 短句为主，不追求完美
- ✅ 有自己的真实感受和经历
- ❌ 不要端着的正式感
- ❌ 不要太像写文章

## 执行流程

### 用户请求示例

```
用户：帮我把这篇文章发小红书

1. 读取用户的文章内容（用户提供 md 文件路径或粘贴内容）
2. 确认平台 → 小红书
3. 根据平台选择对应开头类型（可问用户偏好，也可自己选最合适的）
4. 按照该平台的标准结构扩写
5. 输出内容
```

### 输出格式

```
---
【X/小红书/即刻】内容

[直接发平台的内容，正文完整]
---
字数：XXX 字
标签：#xxx #xxx #xxx（仅小红书/即刻）
平台适配说明：[简短说明为什么这样写]
---
```

- [ ] 互动引导自然？

## 附录：花生风格参考

**花生风格特点：**
- 口语化但不水，有内容密度
- 短句为主，节奏感强
- 有真实经历和细节
- 观点犀利但不居高临下
- 能自嘲，不装

**反面教材（不要这样写）：**
- "在这个AI快速发展的时代，我们需要..."
- "首先，其次，最后..."
- "值得注意的是..."
- 过于正式的报告腔
